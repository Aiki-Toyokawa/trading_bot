## 3. main（60秒定期実行）設定
### 3-0. ファイル編集コマンド
```bash
sudo nano /etc/systemd/system/trading-bot-run.service
sudo nano /etc/systemd/system/trading-bot-run.timer
```

### 3-1. service
`/etc/systemd/system/trading-bot-run.service`
```ini
[Unit]
Description=Trading Bot Run Once
After=network-online.target
Wants=network-online.target

[Service]
Type=oneshot
User=fig-meat
WorkingDirectory=/home/fig-meat/trading_bot
ExecStart=/home/fig-meat/trading_bot/venv/bin/python /home/fig-meat/trading_bot/main.py
```

### 3-2. timer
`/etc/systemd/system/trading-bot-run.timer`
```ini
[Unit]
Description=Trading Bot Timer (every 60s)

[Timer]
OnBootSec=20s
OnUnitActiveSec=60s
AccuracySec=1s
Unit=trading-bot-run.service

[Install]
WantedBy=timers.target
```

### 3-3. 反映・有効化
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now trading-bot-run.timer
```

### 3-4. 確認
```bash
systemctl status trading-bot-run.timer
systemctl list-timers --all | grep trading-bot-run
journalctl -u trading-bot-run.service -n 100 --no-pager
```

## 4. monitor（Web常駐）設定
### 4-0. ファイル編集コマンド
```bash
sudo nano /etc/systemd/system/trading-bot-monitor.service
```

### 4-1. service
`/etc/systemd/system/trading-bot-monitor.service`
```ini
[Unit]
Description=Trading Bot Monitor Web
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
User=fig-meat
WorkingDirectory=/home/fig-meat/trading_bot
ExecStart=/home/fig-meat/trading_bot/venv/bin/python /home/fig-meat/trading_bot/monitor_web.py --host 0.0.0.0 --port 8787
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

### 4-2. 反映・有効化
```bash
sudo systemctl daemon-reload
sudo systemctl enable --now trading-bot-monitor.service
```

### 4-3. 確認
```bash
systemctl status trading-bot-monitor.service
journalctl -u trading-bot-monitor.service -n 100 --no-pager
ss -ltnp | grep 8787
```

## 7. トラブル時の典型
### 7-1. `status=203/EXEC`
- `ExecStart` のパス誤り
- `venv/bin/python` or スクリプトパスを再確認

### 7-2. `can't find '__main__' module in '/home/fig-meat/trading_bot/'`
- `ExecStart` が改行で分断されている
- `ExecStart=` は必ず1行で記述

### 7-3. Webが同一Wi-Fiの他PCから見えない
```bash
# LAN内だけ許可（推奨）
sudo ufw allow from 192.168.1.0/24 to any port 8787 proto tcp

# 参考: 全ネットワーク許可（必要時のみ）
# sudo ufw allow 8787/tcp

hostname -I
```
- ブラウザ: `http://<RasPi_IP>:8787/`
- `--host 0.0.0.0` で起動していれば、Tailscale未接続でも同一Wi-Fi内から閲覧可能。
- ルーターの「AP isolation / クライアント分離」がONだと同一Wi-Fiでも到達できない場合がある。

- `trading-bot-run.timer` が `active (waiting)`
- `trading-bot-monitor.service` が `active (running)`
- `journalctl` に連続エラーがない
- `main.py` 実行結果JSONが定期出力される

## 9. 電源断/再起動時（コンセント抜き差し後）の確認
### 9-1. 事前確認（自動起動設定）
```bash
systemctl is-enabled trading-bot-run.timer
systemctl is-enabled trading-bot-monitor.service
```
- 両方 `enabled` なら、再起動後に自動復旧する。

### 9-2. 起動後の確認
```bash
systemctl status trading-bot-run.timer
systemctl status trading-bot-monitor.service
systemctl list-timers --all | grep trading-bot-run
ss -ltnp | grep 8787
```

### 9-3. ログ確認
```bash
journalctl -u trading-bot-run.service -n 100 --no-pager
journalctl -u trading-bot-monitor.service -n 100 --no-pager
```

### 9-4. 注意
- 物理的な電源断はSDカード破損リスクがあるため、可能なら `sudo shutdown -h now` を推奨。
